# evidence-before-completion Good Output Example

Request: Use `evidence-before-completion` for a evidence before completion lane where the user gave a compact request.

Good response shape:

- Recommended skill: `evidence-before-completion` because the request matches `utility-provider` work.
- Read first: active hub or orchestrator request, current authoritative artifact, only the evidence relevant to this pass.
- Evidence gathered: list exact files, command output, docs, or state artifacts inspected.
- Output: fresh claim-to-evidence checks and proof output appended to workflow-state or the active artifact.
- Residual risk: name what is still unverified and the smallest next check.
- Handoff: test-hub.

This is good because it is anchored to project evidence, limits the claim, and makes the next action executable.
