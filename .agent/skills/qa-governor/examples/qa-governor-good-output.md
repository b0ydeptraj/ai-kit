# qa-governor Good Output Example

Request: Use `qa-governor` for a qa governor lane where the user gave a compact request.

Good response shape:

- Recommended skill: `qa-governor` because the request matches `quality` work.
- Read first: PRD or tech-spec, architecture or story, evidence from tests and reviews.
- Evidence gathered: list exact files, command output, docs, or state artifacts inspected.
- Output: .relay-kit/contracts/qa-report.md.
- Residual risk: name what is still unverified and the smallest next check.
- Handoff: review-hub.

This is good because it is anchored to project evidence, limits the claim, and makes the next action executable.
