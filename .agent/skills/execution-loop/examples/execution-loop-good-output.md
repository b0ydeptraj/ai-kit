# execution-loop Good Output Example

Request: Use `execution-loop` for a execution loop lane where the user gave a compact request.

Good response shape:

- Recommended skill: `execution-loop` because the request matches `developer-support` work.
- Read first: story or tech-spec, project-context, relevant support skills.
- Evidence gathered: list exact files, command output, docs, or state artifacts inspected.
- Output: working code plus test evidence.
- Residual risk: name what is still unverified and the smallest next check.
- Handoff: test-hub.

This is good because it is anchored to project evidence, limits the claim, and makes the next action executable.
