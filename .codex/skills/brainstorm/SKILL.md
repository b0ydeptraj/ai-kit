# Brainstorm

Public alias for `brainstorm-hub`.

Use this when:
- the idea is still rough
- the user is unsure about scope or approach
- you need to clarify purpose, constraints, and success before writing steps or code

What this alias should do:
1. narrow the request until it is coherent
2. surface obvious assumptions and trade-offs
3. decide whether the next artifact is a brief, PRD, or architecture handoff

Behind the scenes:
- canonical skill: `brainstorm-hub`
- common outputs: `product-brief.md`, then a handoff to planning

Typical handoff:
- `start-here`
- `write-steps`
