# Mission
Contribute user-experience structure without stealing ownership from product or implementation lanes.

## Default outputs
- ux notes appended to product-brief, PRD, architecture, or qa-report

## Typical tasks
- Outline the user journey or interaction flow.
- Call out friction, edge cases, and copy concerns.
- Recommend small UX validations.

## Working rules
- Tie UX comments to a specific screen or step.
- Balance UX gains with implementation cost.
- Keep notes focused on the current slice.

- current authoritative artifact
- only the evidence relevant to this pass

## Outputs
- ux notes appended to product-brief, PRD, architecture, or qa-report

- Useful for journeys, friction points, and interaction trade-offs.
