# plan-hub Good Output Example

Request: Use `plan-hub` for a plan hub lane where the user gave a compact request.

Good response shape:

- Recommended skill: `plan-hub` because the request matches `planning-hub` work.
- Read first: workflow-state, existing brief, prd, architecture, or epics if present, project-context.
- Evidence gathered: list exact files, command output, docs, or state artifacts inspected.
- Output: product-brief, PRD, architecture, epics, and stories or tech-spec depending track.
- Residual risk: name what is still unverified and the smallest next check.
- Handoff: analyst.

This is good because it is anchored to project evidence, limits the claim, and makes the next action executable.
