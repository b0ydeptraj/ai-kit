# Build It

Public alias for `developer`.

Use this when:
- planning is done
- files need to change now
- the next move is implementation, not more debate

What this alias should do:
1. implement the scoped change
2. keep evidence tied to the actual acceptance criteria
3. hand off with test and browser proof instead of completion claims

Behind the scenes:
- canonical skill: `developer`
- common outputs: working code, verification evidence, updated workflow state

Typical handoff:
- `ready-check`
- `prove-it`
