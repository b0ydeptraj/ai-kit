# Build It

Public alias for `developer`.

Use this when:
- planning is done
- files need to change now
- the next move is implementation, not more debate

What this alias should do:
1. implement the scoped change
2. start from a failing test or failing reproduction when the behavior is testable
3. keep evidence tied to the actual acceptance criteria
4. keep code, comments, test names, placeholder copy, and sample data plain ASCII; do not add decorative icons, emojis, or odd Unicode unless the product explicitly needs them
5. hand off with test and browser proof instead of completion claims

Behind the scenes:
- canonical skill: `developer`
- common outputs: working code, verification evidence, updated workflow state

Typical handoff:
- `ready-check`
- `prove-it`
