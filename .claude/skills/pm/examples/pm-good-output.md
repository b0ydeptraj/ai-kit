# pm Good Output Example

Request: Use `pm` for a pm lane where the user gave a compact request.

Good response shape:

- Recommended skill: `pm` because the request matches `planning` work.
- Read first: .relay-kit/contracts/product-brief.md or direct scoped request, .relay-kit/contracts/project-context.md.
- Evidence gathered: list exact files, command output, docs, or state artifacts inspected.
- Output: .relay-kit/contracts/PRD.md, .relay-kit/contracts/epics.md.
- Residual risk: name what is still unverified and the smallest next check.
- Handoff: architect.

This is good because it is anchored to project evidence, limits the claim, and makes the next action executable.
