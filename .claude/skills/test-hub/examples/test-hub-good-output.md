# test-hub Good Output Example

Request: Use `test-hub` for a test hub lane where the user gave a compact request.

Good response shape:

- Recommended skill: `test-hub` because the request matches `verification-hub` work.
- Read first: story or tech-spec, implementation evidence, testing-patterns reference.
- Evidence gathered: list exact files, command output, docs, or state artifacts inspected.
- Output: .relay-kit/contracts/qa-report.md, updated workflow-state with pass, fail, or blocked verdict.
- Residual risk: name what is still unverified and the smallest next check.
- Handoff: qa-governor.

This is good because it is anchored to project evidence, limits the claim, and makes the next action executable.
